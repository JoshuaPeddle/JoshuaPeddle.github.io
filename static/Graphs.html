Overview of Graphs





Introduction


     Outside of computer science, our first though when encountering the word graph is usually "bar graph" or "line graph". Another graph and a useful tool for representing data is a structure made up of vertices and edges forming connections between the vertices. Consider an algorithm that outputs an efficient route for digging underground tunnels between buildings. For this example, the buildings would be vertices and the tunnels are the edges.


Traversals


     Visiting each vertex in a graph is the basis of many graph algorithms. Such algorithms are called traversals. BFS (Breadth First Search) and DFS (Depth First Search) are the most popular and usually learned together. Upon initial inspection you'll notice both are similar in implementation and technically accomplish the same thing. (Visit every vertex in a graph) The difference lies in the order that the algorithm considers the vertices.

     BFS selects vertices closer to the source first. This is useful for applications where the target has a higher chance of being closer to the source.


BFS



     DFS moves away from the source by selecting descendants before siblings. If the target tends to be far away for the source BFS will be inefficient because it will spend time checking closer vertices first.

Strongly Connected Components


     Wikipedia defines strongly connected components as "A graph is said to be strongly connected if every vertex is reachable from every other vertex" In a directed graph, it is possible to move from node A to node B down a directed edge and have no route to return to node A. Strongly connected components is used when modeling problems


Example: Truck depot


     Delivery driver. Needs to be able to get back to the depot before running out of gas. One way streets.
